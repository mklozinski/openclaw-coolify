# OpenClaw on Coolify

Deploy [OpenClaw](https://github.com/openclaw/openclaw) on Coolify using pre-built Docker images.

## Services

| Service | Image | Description |
|---|---|---|
| `openclaw` | `coollabsio/openclaw:latest` | OpenClaw gateway with nginx reverse proxy and basic auth |
| `browser` | `coollabsio/openclaw-browser:latest` | Chromium browser sidecar for web browsing capabilities via CDP |

## Quick Start (Coolify)

1. Create a new service in Coolify.
2. Choose **Docker Compose** as the source and point to this repository.
3. Set at least one AI provider API key (e.g. `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `OPENROUTER_API_KEY`).
4. Coolify auto-generates `AUTH_USERNAME`, `AUTH_PASSWORD`, and `OPENCLAW_GATEWAY_TOKEN`.
5. Set your domain вҖ” the exposed port is `8080`.
6. Deploy.

After deploying:

- **OpenClaw UI** вҖ” `https://your-domain` (login with the auto-generated credentials)
- **Browser desktop** вҖ” `https://your-domain/browser/` (for logging into sites that need auth, OAuth, 2FA, captchas вҖ” OpenClaw reuses the session via CDP)

## Persistence

| Volume | Container Path | What it stores |
|---|---|---|
| `openclaw-data` | `/data` | OpenClaw state (`.openclaw/`), workspace, npm packages, Go packages, uv tools |
| `browser-data` | `/config` | Chromium browser profile and session data |

## Updates

The compose file uses `:latest` tags. To update:

1. In Coolify, click **Redeploy** вҖ” Coolify pulls the latest images automatically.
2. Your data persists in the named volumes across redeploys.
